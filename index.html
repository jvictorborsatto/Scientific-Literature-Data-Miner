How it works

📋

Object List

Register compounds, species, genes or any entity of interest, with categories, subcategories and synonyms. Import from CSV in one click.

🔍

Article Mining

Add articles as PDF or paste text manually. The scanner automatically detects which objects appear in each article.

🔎

Search

Cross-search any word, phrase or object name across your entire imported article library. Results open with each matched term highlighted in its own colour directly in the PDF viewer.

📊

Visualization

Generate bar charts, line plots, scatter plots, heatmaps and pie charts from your data. Exportable in high resolution.

🗂️

Combine

Cross-analyze co-occurrences between objects across articles. Build connection maps, compare object groups, and generate synthesis tables — with lazy loading and stale-aware refresh.

💾

Portable Projects

Everything saved in a single .sldm file (SQLite). Easy to share, back up or migrate between computers.

📝

Built-in CSV Editor

View and edit data tables directly inside the app, without opening external spreadsheets.

Typical workflow

1

Create a project

Open SLDM and create a new project. A .sldm file will be created — it stores everything.

2

Register your objects of interest

In the Object List tab, add compounds, species or genes. Include synonyms to improve automatic detection.

3

Import your articles

In the Article Mining tab, add PDFs or paste texts. Use Scan to automatically detect which objects appear.

4

Search across your library

In the Search tab, type any word or phrase and find which articles contain it — with AND/OR logic between terms. Click any result to open the PDF with highlights.

5

Combine and cross-analyze

In Combine, explore co-occurrences between objects, build connection maps and comparison tables across your article collection.

6

Generate visualizations

In the Visualization tab, choose the chart type, configure axes and export the image ready for your paper.

Graphical interface

SLDM uses PyQt5 as its UI framework, with a tab-based main window (QTabWidget). The visual theme is applied globally via QSS.

PyQt5QTabWidgetQSS

Database

All data is stored in a SQLite database embedded in the .sldm file. The access layer (database.py) abstracts all queries.

SQLitesqlite3

Text extraction & mining

PDFs are processed via PyMuPDF (fitz). The scanner finds registered objects and synonyms using normalised regex matching. The Search module uses the same engine with per-term colour-coded highlighting.

PyMuPDFfitzregex

Data visualization

Charts rendered with Matplotlib embedded via FigureCanvasQTAgg. Pandas manages intermediate DataFrames.

User Guide

How to use each tab

SLDM is organized in five main tabs. Click a tab below to see step-by-step instructions and tips.

📋 Object List

This is the foundation of your project. Here you register all the entities — compounds, species, genes, pharmaceuticals, pollutants — that SLDM should look for when scanning articles. The more complete your object list, the better the automatic detection.

1
Add objects manually or via CSV

Click Add Object to create a new entry with name, category, and optionally subcategory. Or go to File → Import Objects CSV to load multiple objects at once.

2
Add synonyms to improve detection

Select any object and click Edit. In the synonyms field, add alternative names separated by comma (e.g. ATZ, 2-chloro-4-ethylamine). SLDM uses all synonyms during scanning.

3
Organize with categories

Use consistent category names (e.g. pesticide, drug, species). Categories are used later to filter and group results in the Visualization and Combine modules.

4
Export the list anytime

Use File → Export Objects CSV to save a copy. Useful to reuse the same list across different projects.

Tips & notes
Synonyms with commasIf a synonym contains a comma, wrap it in double quotes in the CSV: "name A, name B". This avoids column misalignment during import.
Case-insensitive matchingSLDM normalizes text before scanning, so Atrazine, ATRAZINE and atrazine all match the same object.
Reuse across projectsExport your object list as CSV and import it into a new project. This saves time when starting similar reviews.

🔍 Article Mining

This tab is your bibliographic library. Add the articles of your systematic review, then run the automatic scanner to detect which registered objects appear in each article — by PDF, pasted text, or any combination.

1
Add an article

Click Add Article. Fill in metadata manually or paste the article text directly. To load a PDF, click Attach PDF — SLDM extracts the text automatically via PyMuPDF.

2
Fetch metadata from DOI

Paste the DOI and click Fetch Metadata. SLDM queries Crossref and automatically fills in title, authors, journal and year.

3
Scan for objects

Select one or more articles (or all with Ctrl+A) and click Scan. SLDM searches the text for all objects and synonyms from your Object List.

4
Review the results

Click on any article to see which objects were detected, in which paragraphs, and with what frequency. Add or remove detections manually if needed.

Tips & notes
Scan all at onceAfter adding all your articles, press Ctrl+A to select all, then click Scan. SLDM processes each article sequentially — ideal for large batches.
No PDF? Use paste textCopy the abstract or full text from the publisher's website and paste it directly into the article's text field. Scanning works exactly the same way.
Crossref requires internetThe Fetch Metadata function requires an active internet connection. All other features work fully offline.

🔎 Search

A free-text cross-search engine over your entire article library. Type any word, phrase, or object name and find exactly which imported articles contain it — with flexible AND/OR logic between multiple search rows. Results open the PDF with every matched term highlighted in its own distinct colour.

1
Enter your search terms

Type any word or phrase in the search row. Add more rows to combine multiple terms. Choose AND to require all terms, or OR to match any of them.

2
Toggle case sensitivity

Enable the Case-sensitive option when you need exact capitalisation matching (e.g. distinguishing a gene acronym from a common word).

3
Review the matching articles

The results panel lists every article where your terms were found. Each result shows the article title and how many matches were detected.

4
Open highlighted PDF

Click any result to open the built-in PDF viewer. Each search term is highlighted in its own colour across all pages — or the abstract/text if no PDF is attached.

Tips & notes
Searches imported text onlySearch operates on the text already imported into your project — PDFs, pasted abstracts, or raw text. It does not connect to the internet or external databases.
Up to 10 highlight coloursEach search term gets its own highlight colour in the PDF viewer (up to 10 distinct colours). Overlapping highlights use the last-applied colour.
Use Search + Mining togetherRun Article Mining first to identify which objects are present. Then use Search to dive deeper into specific passages, phrases or data values not covered by your Object List.
Case-insensitive by defaultBy default, LC50, lc50 and Lc50 all match. Enable case-sensitive mode only when capitalisation is meaningful for your query.

📊 Visualization

Build publication-ready charts directly from your project data. Choose the data source, chart type, configure axes and filters, and export in high resolution.

1
Choose the data source

Select whether to plot data from Article Mining (citation counts, object co-occurrences) or other compiled data in your project.

2
Select chart type and axes

Choose from bar, line, scatter, heatmap or pie. Configure X and Y axes using the available fields — object name, category, parameter, value, year, journal, etc.

3
Apply filters

Use the filter panel to restrict the data by category, year range, specific objects or parameters. Changes update the chart preview in real time.

4
Export the chart

Click Export to save as PNG or SVG at your desired resolution. Ready for papers, presentations or reports.

Tips & notes
Heatmaps for co-occurrenceThe heatmap type is ideal for visualizing which objects co-occur most frequently across articles. Set X and Y to different object categories for a matrix view.
Export at 300 dpiFor journal submissions, export as PNG with 300 DPI or higher. Use SVG for vector figures that can be resized without quality loss.
Chart configs are savedChart configurations are stored inside the .sldm project file, so you can reopen and tweak them without starting from scratch.

🗂️ Combine

Cross-analyze relationships between objects across your entire article collection. Explore co-occurrences, build visual connection maps, compare object groups side by side, and generate synthesis summary tables — all with performance-optimized lazy loading and stale-aware refresh indicators.

1
Build the analysis pool

Click Refresh to load the pool of articles and object terms. The pool is built once and cached — it will not rebuild automatically on tab switches, keeping performance fast.

2
Search co-occurrences

Select objects and click Search to find which articles mention them together. Results show frequency counts and the list of matching articles.

3
Draw a connection map

Click Draw Map to generate a visual graph of object relationships. Node size reflects frequency; edges show co-occurrence strength.

4
Compare groups and export

Use the Compare and Summary Table sub-tabs to contrast object groups. A ⚠ stale indicator appears if the pool has changed — click ↺ Refresh to update before exporting.

Tips & notes
Stale indicatorA ⚠ icon appears on charts and tables when the underlying pool has changed (e.g. after adding new articles). Always click ↺ Refresh before drawing conclusions from updated data.
Lazy loading for performanceSub-tabs in Combine only load on first visit. Switching between them is instant — computation only reruns when data changes and you explicitly refresh.
Connection Map tipsFor cleaner maps, filter to a single category before drawing. Too many objects at once can produce a dense graph — use the object selector to focus on the most relevant subset.

About the project

SLDM

SLDM was born from the practical need to organize large volumes of scientific literature in systematic reviews of toxicology and ecotoxicology, where manually identifying the presence of hundreds of compounds across dozens of articles is a slow and error-prone process.

The software was developed entirely in Python, with a focus on usability for researchers without a technical programming background. The interface reflects the natural flow of a bibliographic review: register → mine → search → combine → visualize.

Distributed as a standalone executable requiring no environment setup, SLDM can be used immediately on any Windows computer.
